# Cloudflare Tunnel + Traefik Setup Behind CGNAT (Starlink)

This guide describes how to configure **Traefik** with **Cloudflare Tunnel** to host applications behind a CGNAT (such as Starlink). Cloudflare handles HTTPS termination, bypassing Starlink restrictions, while Traefik routes traffic locally to your apps.

---
```
sudo mkdir -p /usr/share/keyrings
curl -fsSL https://pkg.cloudflare.com/cloudflare-public-v2.gpg \
  | sudo tee /usr/share/keyrings/cloudflare-public-v2.gpg >/dev/null

echo "deb [signed-by=/usr/share/keyrings/cloudflare-public-v2.gpg] \
  https://pkg.cloudflare.com/cloudflared any main" \
  | sudo tee /etc/apt/sources.list.d/cloudflared.list

sudo apt update
sudo apt install cloudflared



```
---

## 1. Folder Structure

```
/etc/dokploy/traefik/
├─ traefik.yml             # Static Traefik config
└─ dynamic/
   └─ rpc-gateway.yml      # Dynamic routers/services config

/etc/cloudflared/
├─ config.yml              # Cloudflare Tunnel config
└─ <tunnel-credentials>.json  # Tunnel credentials generated by cloudflared
```

* `/etc/dokploy/traefik/traefik.yml` → Traefik static config
* `/etc/dokploy/traefik/dynamic/rpc-gateway.yml` → Application routing
* `/etc/cloudflared/config.yml` → Tunnel definition
* Tunnel credentials JSON → generated when running `cloudflared tunnel create`

---

## 2. Traefik Static Config (`/etc/dokploy/traefik/traefik.yml`)

```yaml
global:
  sendAnonymousUsage: false

entryPoints:
  web:
    address: ":80"

providers:
  docker:
    exposedByDefault: false
    network: dokploy-network
  file:
    directory: /etc/dokploy/traefik/dynamic
    watch: true

log:
  level: INFO

api:
  dashboard: true
  insecure: false  # disable in production
```

> ⚠️ **Important:** No ACME / Let's Encrypt. Cloudflare handles HTTPS.

---

## 3. Traefik Dynamic Config (`/etc/dokploy/traefik/dynamic/rpc-gateway.yml`)

```yaml
http:
  routers:
    gateway-router:
      rule: Host(`gateway.thebuidl.xyz`)
      service: gateway-service
      entryPoints:
        - web
      middlewares: []

  services:
    gateway-service:
      loadBalancer:
        servers:
          - url: "http://nodemanager-nm-6ksffx:4040"
        passHostHeader: true
```

> Simplified: Traefik only handles internal HTTP routing. Cloudflare terminates HTTPS.

---

## 4. Cloudflared Tunnel Config (`/etc/cloudflared/config.yml`)

```yaml
tunnel: dokploy-tunnel
credentials-file: /home/thebuidldev/.cloudflared/998f72ad-5215-432a-b68e-63b43994af45.json

ingress:
  - hostname: gateway.thebuidl.xyz
    service: http://localhost:80
  - service: http_status:404
```

> Replace the JSON filename with the actual credentials file path.

---

## 5. Run Traefik Container

```bash
docker run -d \
  --name dokploy-traefik \
  --restart always \
  --network dokploy-network \
  -v /etc/dokploy/traefik/traefik.yml:/etc/traefik/traefik.yml \
  -v /etc/dokploy/traefik/dynamic:/etc/traefik/dynamic \
  -v /var/run/docker.sock:/var/run/docker.sock:ro \
  -p 80:80 \
  traefik:v3.6.7
```

---

## 6. Run Cloudflare Tunnel (Test Mode)

```bash
cloudflared tunnel --config /etc/cloudflared/config.yml run dokploy-tunnel
```

Check logs for:

```
INF Registered tunnel connection ...
```

This indicates a successful connection to Cloudflare.

---

## 7. Install Cloudflare Tunnel as a Service

```bash
sudo cloudflared service install
sudo systemctl enable cloudflared
sudo systemctl start cloudflared
```

This ensures the tunnel auto-starts on reboot.

---

## 8. Cloudflare DNS Setup

* Go to **Cloudflare Dashboard → DNS**
* Add record:

| Type  | Name    | Target                         | Proxy      |
| ----- | ------- | ------------------------------ | ---------- |
| CNAME | gateway | `<TUNNEL-ID>.cfargotunnel.com` | 🟠 Proxied |

> Ensure the cloud icon is **orange (Proxied)**, not DNS-only.

---

## 9. Test Your Setup

1. Browser: `https://gateway.thebuidl.xyz` → should load your app with Cloudflare HTTPS
2. Local server test:

```bash
curl -H "Host: gateway.thebuidl.xyz" http://localhost
```

3. Traefik dashboard (if enabled): `http://<server-ip>:8080`

---

## 10. Notes / Best Practices

* Traefik only needs **HTTP (port 80)**; Cloudflare terminates HTTPS.
* Remove or disable ACME / Let’s Encrypt in Traefik.
* Starlink CGNAT no longer blocks your site.
* Cloudflare Tunnel handles reconnections automatically.
* Add more subdomains by updating `config.yml` and Traefik dynamic files.

---

This configuration provides a **plug-and-play setup** for hosting apps behind CGNAT with Traefik and Cloudflare Tunnel, ensuring secure HTTPS and reliable routing without needing a public IP.
